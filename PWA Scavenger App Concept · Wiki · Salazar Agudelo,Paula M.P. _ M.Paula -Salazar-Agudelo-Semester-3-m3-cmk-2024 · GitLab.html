    Comic-Con Scavenger Hunt PWA

    LO2 - Transferable Production

    Objective

    To create and improve the Comic-Con Scavenger Hunt PWA, I focus on:

    • Task Pop-Up Flow: Creating dynamic interactions for users completing tasks.
    • Blip Markers: Designing visual indicators for tasks on the map.
    • Mobile Adjustments: Ensuring the PWA is responsive across devices.

    This work used code methods, often fixing issues and addressing problems such as conflicts caused by combining changes and short time limits.I made the first iteration of the task pop-up and the first iteration of the flow of all the tasks of our concept. Still, after merging conflicts, Levi fixed the issue, combined his part, and made part of the camera logic with Mick; I just focused and made the ofter futures I had to finish in the blip maker show on each part of the map, make the feeling it's a live tracking location.

    Documentation and Version Control

    • Git Repository:
      • Our concept on GitLab: Comic-Con Scavenger Hunt PWA
    • ReadMe in GitLab
    • Branch Management:
      • My contributions were developed on specific map navigation:
        • feature/task-pop-up
        • feature/map-blip-markers
      • Changes were peer-reviewed and merged into the main branch after testing.
    • Commits:
      • Examples of my structured commit messages:
        • feat: added task pop-up animations , design iterations, made media quesrys for the all the task for phone size
        • fix: resolved marker alignment issue on the map and iteration of conlor and design and also make the function of the position of the blip.

    My Contributions

    1. Task Pop-Up Flow

    Process

    The task pop-up feature was designed to provide a blip to orient the map's location of the task solution and indicate to users when they interacted with tasks and the location at the comic-con.

    During development:

    • Event Listeners:
      • Used addEventListener to trigger task pop-ups when users interact with map markers or other UI elements.
      • Dynamically updated the pop-up content ( task description, input fields) based on the task type (camera or text).
    • Dynamic DOM Manipulation: Applied DOM methods to generate and remove pop-ups based on task completion.
    • Feedback Iteration:
      • Initially, the pop-up displayed missing information part of the pop-up because of the size of the phone in Android or iPhone. When you check some examples, we tested and discussed what we should improve, which is implemented in the media query for two different sizes.

    CSS Media Queries:

    • Applied breakpoints to adjust the layout for smaller screens.
    • Ensured elements like pop-ups and markers fit within mobile viewports.

    Testing and Iteration:

    • Tested the app on various devices and browsers to identify and fix layout issues.
    • Iterated on designs based on feedback from users and team members.
    • Feedback revealed the need for more interactivity ( buttons for completing tasks, for example, when the task it is done, show some pop-up if you win or lose and more interaction, which I implemented in subsequent iterations in our Figma.
    • Responsiveness: Adjusted the layout of pop-ups using CSS media queries to ensure usability on mobile devices, I make different iterations of this part of the code.

    Challenges

    • Merge conflicts of the flow or logic of our concept when I pushed my changes. These were resolved to try to fix multiple, and Lindy and Levi also helped with some other iterations to fix this problem.
    • Mobile layout adjustments required multiple iterations to achieve an optimal design by fixing the pop-up design based on group feedback.
    • Overlapping Elements: Adjusted positions and dimensions to prevent elements from overlapping the text and the button on smaller screens to progress to the next page.
    • Time Constraints: Limited testing time is required to prioritize what iteration was more good and interactive engagements for the user when I showed it to another member of the group for feedback improvements.

    2. Blip Markers for the Map

    Process

    The blip markers were designed to visually indicate task locations on the map. My approach included:

    • Static Positioning: Used CSS properties to place blips accurately on the map.
    • Hover Effects: Added CSS transitions for hover states to improve user experience.
    • Team Feedback:
      • I asked the team to review marker placement on the map.
      • We suggested adding interactivity ( displaying descriptions when clicked), but time constraints led us to focus on the core functionality.
    • Iterative Refinement:
      • Adjusted the blip to look like a live location tracker to sizes and positions based on team suggestions and testing results.
      • Improve the task navigation when you win or fail the design to align with project deadlines for Comic-Con Utrecht, but we did not improve in this iteration.

    Challenges

    • Deciding blip marker positions required multiple rounds of feedback and testing.
    • Adding advanced interactivity was deprioritized due to time constraints.

    3. Mobile Adjustments

    Process

    Making the PWA responsive across devices was a key priority. My process involved:

    • CSS Media Queries: Used breakpoints to adjust the layout for different screen sizes.
    • Relative Sizing: Applied percentage-based dimensions and max-width to ensure pop-ups fit on smaller screens.
    • Testing Across Devices: Conducted iterative testing on various mobile devices to identify layout issues and refine the design.

    I make functions that can be used again reusable functions, for example, showPopup() to show and set up pop-ups for tasks based on what type it is (like camera or text), and completeTask() to take care of finishing a task by checking inputs, changing progress, and turning off UI elements like map buttons. The function addBlipsToMap() makes blips and puts them in place using a settings object that has details like title, description, and where they go, making it easy to expand and change things. This way of organizing keeps task logic together, makes things easier to interact with, and helps with maintenance. There are also visual cues like updates on progress and buttons that are turned off for a smooth user experience.

    Challenges

    The first try made things overlap on small screens, so we needed to change where pop-ups were and how big they were. The short time made it hard to test well on all devices.

    Challenges and Resolutions

    1. Merge Conflicts:
      • Problem: Conflicts when I integrate the task pop-up changes with the new iterations cause conflict with some CSS and javascript.
      • Resolution: Collaborated with Lindy and Levi to resolve conflicts and merge the changes.
    2. Feedback Iterations:
      • Problem: Deciding on blip marker positions and functionality required multiple rounds of feedback.
      • Resolution: Incorporated team suggestions to finalize the blip marker position in the map and design.
    3. Mobile Responsiveness:
      • Problem: Initial adjustments caused layout issues on smaller screens.
      • Resolution: Refined media queries and tested iteratively to achieve a clean and functional design. but there are still some issues with iPhone devices.

    Reflection

    This project helped me:

    • Apply dynamic DOM manipulation to create interactive features like task pop-ups.
    • Use CSS transitions and media queries effectively to improve design and responsiveness.
    • Navigate and resolve merge conflicts through team collaboration.

    This project gave me a great chance to look into dynamic DOM changes in blip position, helping me make parts like task pop-ups for new improvements we do in the Figma iteration. By using CSS stuff like transitions and queries for different screens, I made the design better and more responsive, keeping the app usable on many devices. A challenging but rewarding part was dealing with merge problems, needing good teamwork to mix our work nicely. During this time, I tried to balance feature difficulty and ease of use, focusing on main functions to meet short timelines. Feedback from others was essential for improving my code, allowing me to keep making the app function and look better. These experiences not only made my tech skills better but also helped me learn how to work well in a team in a busy development setting.
